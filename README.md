# 열매똑똑 해커톤 본선 심사 앱

[![GitHub Pages](https://img.shields.io/badge/deploy-GitHub_Pages-181717?logo=github)](https://pages.github.com/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

서울특별시사회복지사협회 · 사랑의열매 서울사회복지공동모금회 공동주관 **열매똑똑 해커톤 본선 심사** 진행을 위한 단일 HTML 웹앱입니다.

심사위원 3인이 각자 태블릿/노트북에서 동시에 채점하고, 점수·심사의견·서명이 자동으로 Google 시트에 기록되며 서명 이미지는 별도 Drive 폴더에 PNG로 보관됩니다. 운영자는 관리자 화면에서 종합 순위표와 서명이 포함된 통합 PDF를 한 번에 내려받을 수 있습니다.

## 🚀 GitHub Pages 배포 (3분)

### 1. 저장소 만들기

```bash
# 새 저장소 생성 후
git clone https://github.com/<본인계정>/<저장소명>.git
cd <저장소명>

# 이 패키지의 모든 파일 복사 후
git add .
git commit -m "초기 배포"
git push origin main
```

### 2. GitHub Pages 활성화

1. 저장소 → **Settings** → 좌측 메뉴 **Pages**
2. **Source**: `Deploy from a branch`
3. **Branch**: `main` / `/(root)` 선택 → **Save**
4. 1–2분 후 `https://<본인계정>.github.io/<저장소명>/`에서 접속 가능

### 3. 심사위원에게 URL 공유

배포 완료된 URL을 심사위원 3명에게 전달하면 즉시 사용 가능합니다. **웹앱 URL은 이미 코드에 내장되어 있으므로 별도 설정 불필요.**

## 🚀 사용법

### 본선 당일 심사위원

1. 운영자가 공유한 GitHub Pages URL 접속
2. 첫 화면에서 본인 이름 선택 (드롭다운)
3. **공정 심사 서약** 5개 항목 모두 체크 → 시작
4. 작품 드롭다운에서 평가할 작품 선택
5. 점수 입력 (직접 숫자 또는 ± 버튼) → 의견 작성 → **서명**
6. **저장 + 시트 전송** 버튼 → 즉시 시트와 Drive에 반영
7. 9팀 모두 평가 후 종료

### 운영자 (관리자)

1. 첫 화면 하단 **"관리자 모드 →"** 클릭 → 비밀번호 `admin`
2. 실시간으로 모든 심사위원의 진행 상태와 순위표 확인
3. **🔄 시트에서 불러오기**: 다른 기기에서 채점된 데이터를 동기화
4. **📥 CSV 내려받기**: 엑셀/숫자에서 열 수 있는 CSV
5. **📄 통합 PDF (서명 포함)**: 표지 + 종합순위 + 작품별 채점지(서명 포함) 한 번에

## 📊 시트 구조 (자동 생성)

Apps Script가 첫 저장 시 자동으로 "심사결과" 탭을 생성합니다.

| 컬럼 | 헤더 |
|---:|---|
| A | 저장시각 |
| B | 심사위원번호 |
| C | 심사위원성함 |
| D | 심사위원소속 |
| E | 팀ID |
| F | 앱명 |
| G | 제출기관 |
| H | 제출자 |
| I | 부문 |
| J | 문제정의명확성(25) |
| K | 실행완성도(25) |
| L | 개선효과설득력(25) |
| M | 확산가능성(25) |
| N | 감점(-10~0) |
| O | 총점(100) |
| P | 심사의견 |
| **Q** | **서명링크** ← Drive 공유 URL |
| **R** | **서명시각** |

같은 심사위원이 같은 작품을 다시 저장하면 기존 행이 **덮어쓰기**됩니다.

## 🖼️ 서명 파일 (Drive)

```
signature_J{심사위원번호}_{팀ID}_{타임스탬프}.png
```

예시: `signature_J1_A1_20260427_143205.png`

총 27개 파일이 생성됩니다 (3 심사위원 × 9 작품). 같은 심사위원+작품에 다시 서명하면 이전 파일은 자동으로 휴지통으로 이동합니다. 공유 권한은 **링크가 있는 사람 누구나 보기**로 자동 설정됩니다.

## 🔧 Apps Script 재배포 (필요한 경우만)

웹앱 URL을 새로 발급받아야 할 때만 진행:

1. <https://script.google.com> → 기존 프로젝트 열기 (또는 새로 만들기)
2. `apps-script/Code.gs`의 내용을 붙여넣기
3. 함수 선택 → `testFolderAccess` → ▶ 실행 (권한 동의)
4. **배포 → 새 배포** → 유형: **웹앱**, 액세스: **모든 사용자** → 배포
5. 새 URL 복사
6. `index.html`의 `DEFAULT_WEBAPP_URL` 상수를 새 URL로 교체
7. `git commit && git push`

> 한 번 배포된 후에는 **"새 배포"가 아닌 "배포 관리 → 수정"**으로 같은 URL을 유지하며 코드만 업데이트할 수 있습니다.

## 🔧 문제 해결

| 증상 | 해결 |
|---|---|
| 서명링크에 "업로드 실패" 표시 | Apps Script 실행 계정에 Drive 폴더 편집 권한이 없음. `testFolderAccess` 실행으로 권한 동의 |
| 시트 컬럼이 16개에서 변하지 않음 | "심사결과" 탭을 통째로 삭제하면 새 헤더로 자동 재생성 |
| 통합 PDF의 서명이 빈 칸 | 서명 데이터는 그 기기의 localStorage에 보관됨. 운영자 기기에서 채점부터 진행하거나, 각자 본인 기기에서 PDF 받기 |
| CORS 에러 | Apps Script 배포 시 액세스 권한이 "모든 사용자"인지 확인 |
| localStorage 날아감 | 시크릿 모드 사용 금지. 관리자 화면 → 🔄 시트에서 불러오기로 점수는 복구 가능 (서명 이미지는 재서명 필요) |
